Το PaperPilot είναι ένα Agentic RAG σύστημα επάνω σε 50 ArXiv papers από τους τομείς NLP, LLMs, RAG και AI Agents (2020-2026). Χρησιμοποιεί LangGraph multi-agent graph (Planner, Researcher, Synthesizer κόμβοι) με δύο εργαλεία - τοπική RAG ανάκτηση (rag_retrieve) και live ArXiv αναζήτηση (arxiv_search) - επιτρέποντάς του να απαντά σε ερωτήσεις εντός corpus με τεκμηριωμένα στοιχεία και να αρνείται χαρακτηρισμένα όταν η ερώτηση είναι εκτός πεδίου.

Αξιολογούμε τρεις εκδόσεις pipeline σε ένα golden set 84 ερωτήσεων με: RAGAS (Context Precision/Recall, Faithfulness, Answer Relevancy), Tool Call Accuracy (agentic μετρική) και τη HAIC suite (Helpfulness, Trust Calibration, Effort Saved, Harm Potential). Η v2 ξεπερνά τη v1 σε 9 από 13 διαστάσεις. Η v3 (LangGraph multi-agent + diversity reranking) βελτιώνει Context Recall +25.4% και HAIC Judge Score +28.8% vs v2.

4b Golden Set - Κατασκευή, Δομή & Αξία

Το golden set είναι η βάση κάθε αξιολόγησης: 84 ερωτήσεις με προκαθορισμένη αναμενόμενη απάντηση και αναμενόμενη αλυσίδα εργαλείων. Χωρίς αυτό, οι μετρικές RAGAS, TCA και HAIC δεν μπορούν να υπολογιστούν. Παρακάτω περιγράφεται αναλυτικά πώς κατασκευάστηκε, γιατί έχει αυτή τη δομή και τι εγγυάται για την εγκυρότητα της αξιολόγησης.

Στάδιο 1 - LLM-Assisted Candidate Generation

Η κατασκευή ξεκίνησε με αυτόματη παραγωγή υποψήφιων ερωτήσεων από τα ίδια τα papers του corpus. Το script src/paperpilot/eval/golden_gen.py εκτελεί την εξής διαδικασία:

  1. Φορτώνει όλα τα parsed papers και δημιουργεί chunks με SectionAwareChunker
  2. Φιλτράρει chunks από sections abstract, method, results, experiments, introduction (αποφυγή references/appendix)
  3. Για κάθε candidate: επιλέγει τυχαία ένα chunk και μια κατηγορία (round-robin στις 6)
  4. Καλεί gpt-4.1-mini (temperature=0.4) με structured prompt που απαιτεί JSON output
  5. Αποθηκεύει candidate με: question, expected_answer, must_cite, category, source_paper_id, source_section
Generation Prompt - Κανόνες Το generation prompt επιβάλλει: (α) για numerical — ο αριθμός πρέπει να υπάρχει στο excerpt, (β) για out_of_context — να είναι NLP-related αλλά αναπάντητο από το excerpt, (γ) για attribution — να ρωτά "ποιο paper / ποιος πρότεινε X", (δ) self-contained ερωτήσεις (χωρίς "in this paper"). Αυτοί οι κανόνες εξασφαλίζουν testability.

Στάδιο 3 - Out-of-Context Augmentation (59 OOC Questions)

Οι 59 OOC ερωτήσεις προστέθηκαν χειροκίνητα σε 4 υπο-κατηγορίες:

Γιατί 59 OOC από 84; Σε πραγματικά συστήματα, η πλειοψηφία των ερωτήσεων χρηστών είναι εκτός corpus scope. Με 59/84 OOC (70%) δοκιμάζουμε την ικανότητα του agent να αρνείται κατάλληλα χωρίς hallucination. Αυτό όμως dilutes τα overall RAGAS scores - γι' αυτό αναφέρουμε πάντα in-context vs overall.

Αξία του Golden Set ως Evaluation Benchmark

Determinism Κάθε ερώτηση έχει συγκεκριμένη expected_answer. Τα RAGAS metrics (CP, CR, Faithfulness, AR) υπολογίζονται βάσει αυτής. Αυτό επιτρέπει αναπαραγώγιμη σύγκριση v1/v2/v3 χωρίς ανθρώπινη κρίση κατά τη μέτρηση.
TCA Scoring Το expected_tools field επιτρέπει Tool Call Accuracy: score=1.0 αν ο agent κάλεσε τα σωστά tools, 0.5 αν σωστά αλλά λάθος σειρά, 0.0 αλλιώς. Αυτό είναι μοναδική agentic μετρική που δεν υπάρχει στο vanilla RAGAS.
Multi-version Comparability Το ίδιο golden set τρέχει σε v1, v2, v3 χωρίς αλλαγές. Οι διαφορές στα scores αντικατοπτρίζουν αποκλειστικά τις αλλαγές στο retrieval pipeline.
SQLite Cache RAGAS και HAIC judge calls cached με SHA-256 key. Δεύτερη εκτέλεση κοστίζει 0 API calls. Η cache επιτρέπει debugging και comparison runs χωρίς επιπλέον OpenAI κόστος.

7 HAIC - Human-AI Collaboration Benchmarking

7.0 Τι είναι το HAIC και γιατί LLM-as-Judge

Το HAIC (Human-AI Collaboration) framework μετρά την αξία ενός AI συστήματος από τη σκοπιά της ανθρώπινης συνεργασίας: πόσο βοηθά, πόσο εμπιστεύεται ο χρήστης, πόσο γρήγορα αποκρίνεται. Στο PaperPilot, η αξιολόγηση είναι πλήρως αυτοματοποιημένη: δεν υπάρχουν πραγματικοί χρήστες - ο ρόλος του ανθρώπου προσομοιώνεται από LLM judge.

Γιατί όχι πραγματικοί αξιολογητές; Human annotation για 84 ερωτήσεις × 3 εκδόσεις = 252 ανεξάρτητες κρίσεις. Απαιτεί domain expertise (NLP/RAG γνώση), εκτενή briefing και ώρες εργασίας. Ο LLM judge (gpt-4.1-mini, temperature=0) παράγει deterministic κρίσεις, cached μέσω SQLite, με κόστος περίπου $0.02 για ολόκληρο το golden set. Η εγκυρότητα LLM-as-judge έχει τεκμηριωθεί στη βιβλιογραφία (RAGAS paper, Zheng et al. 2023 "Judging LLM-as-a-Judge").

7.1 Workflow Αυτοματοποιημένης Αξιολόγησης

Για κάθε ερώτηση του golden set, εκτελείται το εξής pipeline:

1

2
Agent Run

3

4

5

6

7.2 Judge Prompts - Ακριβής Υλοποίηση

Χρησιμοποιούνται δύο ξεχωριστά prompts - ένα για in-context ερωτήσεις και ένα για OOC. Η διαχωριστική λογική: για OOC, η "σωστή" απάντηση είναι η άρνηση, οπότε ένα helpfulness-based prompt θα τιμωρούσε τη σωστή συμπεριφορά.

In-Context Judge Prompt (Helpfulness 1-5) 

You are a strict HAIC (Human-AI Collaboration) evaluator. Score the AI answer below.

Metric: helpfulness
Scale: 1-5 (1=useless, 5=fully addresses the user's intent)
Description: Does the answer give the user what they actually need to act/decide?

User question: {question}
Reference answer (may be approximate): {reference}
AI answer: {answer}

Return strict JSON: {"score": <integer 1-5>, "reason": "<one sentence>"}

OOC Judge Prompt (Refusal Quality 1/3/5) 

You are evaluating whether an AI assistant correctly handled an out-of-corpus question.

The question is OUTSIDE the AI's knowledge base (NLP/LLM/RAG papers 2020-2026).
A correct response REFUSES or says the topic is not in its corpus.
An incorrect response invents an answer (hallucination).

Score:
  5 = Agent clearly states it cannot answer / topic not in corpus (correct refusal)
  3 = Agent partially hedges but still provides some content (partial hallucination)
  1 = Agent confidently answers as if it knows (full hallucination)

User question: {question}
AI answer: {answer}

Return strict JSON: {"score": <integer 1, 3, or 5>, "reason": "<one sentence>"}
Αντι-hallucination κίνητρο στο OOC prompt Αν χρησιμοποιούσαμε το helpfulness prompt για OOC, ένας agent που "απαντά με αυτοπεποίθηση" θα έπαιρνε score=5 ακόμα και αν hallucinate. Το OOC prompt αντιστρέφει αυτό: η σωστή άρνηση παίρνει 5/5. Αυτό ευθυγραμμίζει τη μετρική με την επιθυμητή συμπεριφορά.

7.3 SQLite Caching - Determinism & Zero Rerun Cost

Κάθε judge call cached με key = SHA-256(model + prompt). Η απάντηση αποθηκεύεται ως JSON στο data/cache.db (namespace: "judge"). Δεύτερη εκτέλεση του HAIC eval: 0 API calls, 0 κόστος, ταυτόσημα αποτελέσματα. Αυτό κρίσιμο για debugging: μπορείς να τρέξεις ξανά χωρίς να ανησυχείς για variability.

7.4 HAIC Metrics - Ορισμοί & Φόρμουλες

ΜετρικήΦόρμουλαΕρμηνείαΤιμή v3
EL
Effort Loss		 (t_actual - baseline_s) / baseline_s Ποσοστό χρόνου που "χάθηκε" vs baseline. EL=0 → ίσος χρόνος. EL<0 → AI πιο γρήγορο. 0.0
Tr
Trust		 accepted / (accepted + rejected) Ποσοστό αποδοχής. Προσομοιώνει χρήστη που αποδέχεται ή απορρίπτει κάθε απάντηση. 0.786
HCL
Human Cognitive Load		 1 - clip(mean_latency / RT_MAX, 0, 1)
RT_MAX = 30s		 Πόσο χαμηλό είναι το cognitive load. 1.0 → κάθε απόκριση εντός 30s. 0 → πάντα αργό. 0.428
F
Interaction Freq.		 events / session_duration_min Πόσα events (query/respond/accept/reject) ανά λεπτό. Υψηλό F → active session. —
A
Adaptability		 tanh((Tr_late - Tr_early) / Tr_early) Βελτίωση Trust μεταξύ αρχής και τέλους session. A>0 → χρήστης εμπιστεύεται περισσότερο. —
EffScore
EfficiencyScore		 1 / (1 + EL) Σύνθετο score: 1.0 → μηδενικό effort loss (AI δεν επιβραδύνει τον χρήστη). 1.0
Γιατί EfficiencyScore=1.0 στο v3; Ο χρόνος baseline υπολογίστηκε ως 89s/ερώτηση (125 λεπτά / 84 ερωτήσεις). Ο πραγματικός χρόνος περιλαμβάνει: 2s inter-question pause + ~30s agent response + 8s simulated human decision ≈ 40s/ερώτηση. Το 40s < 89s → EL < 0 → EfficiencyScore > 1, clamped στο 1.0. Αυτό δεν σημαίνει τέλεια απόδοση - σημαίνει ότι ο agent είναι ταχύτερος από τη manual baseline.
